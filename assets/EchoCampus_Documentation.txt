EchoCampus
Meet. Learn. Build.

1. Objective
EchoCampus provides a single digital platform for campus communication and utility flows:
announcements, complaints, marketplace trading, lost-and-found, faculty directory, and
anonymous student chat.

2. Current Product Scope
- Role-based experience for students, faculty, and admin users
- Supabase-backed APIs for core modules
- Firebase anonymous chat channel for real-time messaging
- Dashboard-first navigation for role-specific workflows

3. Feature Summary
- Announcements: faculty posting, authenticated read access
- Complaint System: student complaint posting with upvote toggle support
- Marketplace: student-driven listing flow and owner-only "mark sold"
- Lost and Found: report item + contact details + optional image
- Faculty Directory: searchable faculty cards
- Anonymous Chat: random session-code based identity labels

4. Technology Stack
- Frontend: Next.js App Router + React 19 + TypeScript + Tailwind CSS
- Backend APIs: Next.js route handlers
- Database/Auth: Supabase
- Real-time chat: Firebase (anonymous auth + Firestore)

5. Access Control Model
- User role source: `users.role` (`student`, `faculty`, `admin`)
- Faculty signup/source: emails present in `directory` are treated as faculty accounts
- Protected route prefixes:
  - `/main/student/*`
  - `/main/faculty/*`
- `admin` is handled as faculty-like access in app guards.
- Write restrictions are enforced in Supabase RLS policies.

6. Usage Limits (Database Trigger Layer)
- Complaints: 1 complaint per user per 7 days
- Marketplace: 1 listing per user per 3 days
- Lost and Found: up to 2 posts per user per 24 hours
